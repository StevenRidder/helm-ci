## 4. 🔴 Materially-false `Done` claims (6)

1. **CONTRACT-10 — alarm reliability tier: server half absent.** Client decode/ack machinery is
   built (`web/nav-client.js:138-267`), but `helm_server.cpp` emits no alarm frames, no persist, no
   resend-until-ACK, no `alarm.ack` handler. `docs/CONTRACT-ALARM-SCHEMA.md:183` confirms.
   **Cascade:** ALARM-1…11 and AIS-5 ride "the CONTRACT-10 schema" — they run fine as *client-side*
   logic (tested 52/52) but the reliable-delivery / off-boat promise does not exist.
2. **CONTRACT-12 — "one TLS origin": no TLS in the binary.** No SSL/cert/HTTPS anywhere in
   `helm_server.cpp`; plain `ws://`+`http://`. `server-endpoint.js:52` mirrors the page scheme, so
   `wss://` works *behind an external proxy that isn't part of this repo*.
3. **CONTRACT-6 — lastSeq resume ignored.** Client sends `lastSeq` on reconnect
   (`nav-client.js:348`); server `parse_sub()` never reads it → full re-snapshot every reconnect.
4. **WX-11 — ensemble spread is synthetic.** Both members in `web/data/wxtiles/ensemble.json` are
   labeled `"synthetic, NOT FOR NAVIGATION"`; the spread math (`cog.js:347-427`) runs over demo
   noise. No real GFS/ECMWF feeds. Cannot be "fixed" offline — must be re-scoped/relabeled.
5. **OWNSHIP-6 — head-up rotates to a fabricated heading, silently.** Vesper HDG is ignored (engine
   parses only HDT), so head-up uses `sim_hdg` (route bearing ± jitter) with no warning
   (`ownship.js:207-215`, `helm_server.cpp:1422`). Violates the fail-loud honesty policy.
6. **AIS-2 — symbology breaks the tap card.** When `helm-ais-symbol` replaces `ais-vessels`, the
   click/cursor handlers stay bound to the hidden layer (`index.html:1003-1005`) → AIS targets
   become un-clickable on the chart. The list `openCard()` is a workaround, not a map fix.

  BOAT-1 → ROUTING gate on this.

  per-domain-router ownership design PLACES/AI rely on.

  server (only in `helm_server`); chart-groups have a `?cells=` filter but no management UI.

  no-ops (only YDWG-over-IP and aisstream.io work). **CONN-5** — `HELM_SIGNALK` env still bypasses
  the Connections UI (`helm_server.cpp:1777`).
- **WX-10** — GFS real; ECMWF/ICON are stubs (exit 3). **WX-7** — route ribbon needs the (down)
  :8090 backend. **WX-13** — math real + tested + panel registers; no real apparent-wind source, so
